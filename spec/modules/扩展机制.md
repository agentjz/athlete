# 扩展机制

## 当前扩展口

Athlete 当前有 3 个标准扩展口：

1. tools：负责动作
2. skills：负责按需 workflow 与知识
3. MCP：负责接外部能力源

## web-research / browser-automation 的边界

这两个 skill 现在已经是真实 workflow，而不是占位文档。

职责边界：

- `web-research`
  - 何时先搜
  - 何时打开网页
  - 何时提取信息
  - 何时总结
  - 何时落文档
- `browser-automation`
  - 何时导航
  - 何时 snapshot
  - 何时 click / type
  - 何时 screenshot
  - 何时做网页交互

不负责：

- 实现浏览器 transport
- 取代 MCP / tool

## 联网能力分层

当前分层是：

1. MCP / tools
   - 提供真实浏览器能力
   - 当前优先是官方 `@playwright/mcp`
2. workflow skill
   - 提供使用策略
   - 当前是 `web-research` 与 `browser-automation`

设计原则：

- 联网能力先做成 `tool` 或 `MCP`
- `skill` 只做 workflow
- 不把底层联网继续堆进 `systemPrompt`

## 当前约束

当前扩展仍必须满足：

- 不破坏 continuation / compact / recovery
- 不让主循环继续长胖
- 不新造平行技能体系
- 不让 MCP 绕开统一 registry
