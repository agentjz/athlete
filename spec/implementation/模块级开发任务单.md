# 模块级开发任务单

## 当前阶段：多 Agent 调度系统收口

### 目标

建立一个正式的多 Agent 调度系统，让 `lead / subagent / teammate / background` 的 split / dispatch / wait / merge 都靠账本事实推进，而不是靠 prompt 记忆。

### 必做任务

1. 明确并落盘最小调度图模型：
   - objective key
   - orchestrated task kind
   - executor kind
   - dependency / join 关系
2. 保持正式真相源仍然收口在 SQLite 账本，不回退到 JSON。
3. 扩现有 orchestrator：
   - split 规则
   - dispatch 规则
   - wait condition
   - merge / join 推进
4. 让 readiness / ownership / handoff legality / worktree legality 从正式账本派生。
5. 让 reload / continue / reconcile 后能恢复：
   - 谁在做什么
   - 哪些必须等
   - 哪个 merge 节点该 ready
6. 清理与新调度模型冲突的旧残余路径。
7. 先补失败测试，再实现，再做验证。

## 本阶段验收

1. 复杂 objective 能落成机器可识别的最小任务图。
2. lead 不会重复 dispatch 已经在跑的 teammate / background work。
3. background child 完成后，后续 lead / merge step 会正式推进。
4. 需要等待时，调度动作会进入 wait，而不是继续乱派工。
5. 子任务完成后，父任务能进入 merge 或完成。
6. reload 后仍能恢复 active owner / running job / bound worktree / next wait reason。
7. 旧 prompt 型协调不再承担裁决。
8. `npm.cmd run check` 与 `npm.cmd test` 通过。

## 本阶段明确不做

1. 不重做第 5 轮 SQLite 账本迁移。
2. 不把 session / checkpoint / runtime stats 一起迁到 SQLite。
3. 不新增 UI、通道或平行 orchestrator。
4. 不为错误旧调度路径做长期双轨兼容。
