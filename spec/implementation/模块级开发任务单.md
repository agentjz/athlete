# 模块级开发任务单

## 当前阶段：SQLite 控制面账本收口

### 目标

把 `task / team / protocol request / background / worktree` 的正式真相源收口到一个 SQLite 本地账本里，并清理旧 JSON 真相源。

### 必做任务

1. 在 `.athlete/control-plane.sqlite` 建立正式账本。
2. 拆分账本层：
   - 实体结构
   - 建表 / 迁移
   - 查询 / 写入
   - 领域 store
3. 保持 `TaskStore / TeamStore / ProtocolRequestStore / CoordinationPolicyStore / BackgroundJobStore / WorktreeStore` 的上层语义稳定。
4. 把 orchestrator、runtime summary、reconcile、reset 等读取路径切到统一账本。
5. 保留 JSONL 审计流，但禁止其继续参与机器裁决。
6. 下线旧 JSON 真相源并清理残余：
   - `.athlete/tasks/`
   - `.athlete/team/config.json`
   - `.athlete/team/policy.json`
   - `.athlete/team/requests/`
   - `.athlete/team/background/`
   - `.athlete/worktrees/index.json`
7. 先补失败测试，再实现，再做验证。

## 本阶段验收

1. SQLite 账本可以在干净目录完成初始化。
2. 重载后 task / team / request / background / worktree 状态一致。
3. 并发 claim / 绑定由账本裁决，不再由 JSON 读写时序碰运气。
4. orchestrator 读取的是统一真相，不是文件残影。
5. 旧 JSON 不再双写，也不再被当作正式状态读取。
6. `npm.cmd run check` 与 `npm.cmd test` 通过。

## 本阶段明确不做

1. 不把 session / checkpoint / runtime stats 一起迁到 SQLite。
2. 不保留旧 JSON 的长期兼容层。
3. 不为旧状态做双写过渡。
4. 不顺手重写整个 runtime。
