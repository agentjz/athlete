# 用户审阅

这一层是给人类看的。

它默认读者不是写代码的人，而是：

- 决定项目方向的人
- 审阅实现是否跑偏的人
- 只看文档、不看实现细节的人

## 你在这里应该看到什么

你应该看到的是：

- Athlete 的核心定位
- 它服务什么样的用户
- 它当前做什么、不做什么
- 当前重要专项为什么值得做、边界在哪里
- 它为什么强调耐跑、总指挥、拆任务、派工和扩技能
- 哪些原则是整个项目不能越线的

你不应该在这里被迫理解：

- 具体文件怎么 import
- 哪个 store 的字段叫什么
- 某个通道的细小接线细节

这些内容都在 `技术实现/`。
