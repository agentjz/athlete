# P06 上下文要能压缩

## 原则

上下文一定会满，所以必须有压缩和续跑机制。

## 为什么

如果没有压缩，长任务跑着跑着就会直接死掉。

## 在 Kitty 里的含义

- `contextRuntime` 统一管理当前上下文、同 session 简报、当前任务工作记忆、压缩、工具进度和请求诊断
- request context 只压缩当前 work surface，不恢复旧的跨任务压缩记忆
- 同 session 简报提供用户连续性，但必须短、结构化、可审阅
- 当前任务工作记忆提供执行连续性，但必须绑定当前 objective
- 原始历史是证据库，不是自动注入的聊天记录
- tool step 到上限后通过 runtime continuation 续跑
- teammate loop 可以在多轮里继续同一任务

这条原则直接保护 Kitty 当前最强的“耐跑能力”。
