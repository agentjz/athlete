# P21 证据不能伪装成完成

## 原则

交付必须对真实证据负责，但机器层不能把验证账本变成自动收口门。

## 为什么

“写了文件”不等于“做成了”。同样，“机器看到改了文件”也不等于“机器知道下一步必须验证”。

Deadmouse 的边界是：

- 模型判断任务是否需要验证、验证是否足够、是否还能收口
- 机器记录验证尝试、命令结果、产物路径和失败事实
- 机器阻止伪造通过、缺失记录、非法状态
- 机器不自动要求 fix、re-verify、pause awaiting user 或继续下一轮

## 在 Deadmouse 里的含义

closeout 不能靠空话。Lead 应该基于真实文件、命令、接口、页面、artifact 或其他证据作判断。

verification state 是账本，不是大脑。它可以显示 not attempted / passed / failed 事实，但不能替 Lead 决定：

- 是否必须继续
- 是否必须重新验证
- 是否必须问用户
- 是否必须换路线
- 是否禁止最终回复

如果用户或任务契约明确要求验证，Lead 必须负责满足该契约；机器层只保存证据并暴露事实。
