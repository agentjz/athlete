# P00 大模型只需要当前目标

## 原则

当下就是最重要的事情。

大模型不需要被塞满记忆，只需要当前目标。

机器层只守死边界，模型层负责活判断。

## 为什么

如果运行时把旧目标、旧 todo、旧 checkpoint、旧队友状态和旧对话细节都塞进 prompt，模型拿到的不是一张干净工单，而是一间堆满旧会议纪要的房间。

这会让模型误以为旧任务仍是当前任务，也会让它在不该继承的上下文里继续上一轮。

## 在 Deadmouse 里的含义

- 当前 prompt 只服务当前目标。
- 旧目标、旧任务板、旧 checkpoint、旧 execution、旧 inbox 和旧协议状态只留在机器账本里，不作为当前目标的正文记忆灌给模型。
- 机器层默认只向模型暴露当前目标和当前目标相关的运行事实。
- 上一轮对话结束后，最终回复和工具证据只进入持久档案，不自动进入下一轮 prompt。
- prompt 不主动携带历史入口，不提醒模型回顾历史；历史入口只作为普通只读工具能力存在。
- 机器层可以判断 objectiveKey 是否一致、tool call 是否闭合、DeepSeek 协议字段是否完整、执行是否仍属于当前目标。
- 机器层不能替模型判断“这个旧任务语义上是否重要”，也不能用关键词替用户或 Lead 决定路线。
- 机器账本完整保留；模型如确实需要，会通过只读历史、session、artifact、event、change record 工具自行查。

## 当前对应

- TurnFrame / Current Objective
- request context 压缩只压当前轮上下文
- 无 turn-end 续写注入；无跨轮 final output 注入；无跨轮 model compression 注入
- session / artifact / event / change record 只读查询工具
- checkpoint 只做机器状态，不做当前目标正文
- task / execution / background 的 objectiveKey 隔离
- DeepSeek reasoning_content 协议回放
