# 产品定义

## Athlete 是什么

Athlete 是一个终端优先的 AI harness。

它不是“在命令行里聊天”，而是一套让模型持续做任务的运行系统：

- 能调用工具
- 能持久化任务状态
- 能续跑长任务
- 能把慢操作放后台
- 能按需调用子智能体或队友

## 当前核心定位

当前 Athlete 的核心定位是：

**一个耐跑、可续跑、面向复杂任务的终端智能体框架。**

它现在最强的价值不是“最会聊天”，而是：

- 不容易中途断掉
- 出错后会继续排查
- 长任务能自动续跑
- 多步任务有状态落盘
- 任务派工和恢复不靠 prompt 记忆

## 下一阶段定位

下一阶段 Athlete 的定位是：

**一个以耐跑主 Agent 为核心、能够统筹任务与技能扩展的总指挥型智能体平台。**

关键词：

- 主 Agent 很强
- 复杂任务时会拆分与调度
- skills 是标准扩展口
- 多 Agent 是按需能力，不是默认军团
- 控制面知道任务为什么 ready / blocked / active / completed

## 目标用户

### 当前主要用户

- 会在终端里工作的开发者
- 想研究 agent harness 架构的维护者

### 未来扩展用户

- 需要多步自动化的人
- 需要文档、表格、资料整理等复杂任务的人
- 想给 agent 增加行业技能的人

## 非目标

- 不做拖拽式工作流平台
- 不做纯聊天机器人
- 不追求“默认一堆 agent 同时乱跑”
- 不把长期人格记忆作为当前阶段的核心卖点
