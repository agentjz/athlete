# v0 范围

## 当前已在范围内

### 运行时

- 单 turn 主循环
- 自动 continuation
- 上下文压缩
- verification 状态机

### 工具与执行

- 文件读写与补丁修改
- shell 执行
- docx / spreadsheet 基础能力
- background job

### 控制面

- 任务板持久化
- teammate 名册与 inbox
- 协议请求与审批状态
- worktree 绑定与隔离

### 扩展

- 项目指令发现
- skill catalog 发现与按需加载
- MCP 工具接入

## 当前不在范围内

- 图形化编排平台
- 跨项目长期人格记忆
- 全自动自治 swarm
- 复杂经济系统、积分系统、市场化调度
- 大而全行业平台能力

## 下一阶段新增范围

1. 主 Agent 的总指挥层
2. 更清晰的任务拆分与派工规则
3. skills 的标准声明与加载协议
4. reviewer / verifier / planner 这类专门角色的演进边界

## 范围控制规则

如果一个新需求同时满足下面两条，应该延后：

- 会削弱当前“耐跑底盘”
- 不能明显增强“总指挥层”或 “skills 扩展口”

当前阶段优先做“更会组织”，不是“看起来更全能”。
