# 测试策略

## 当前目标

测试不是覆盖率装饰，而是用来保护主干结构。

## 当前优先保护对象

1. 主 Agent 耐跑能力
2. 总指挥调度语义
3. 控制面真相源
4. 宿主运行边界
5. 配置治理
6. 宿主行为回归

## 当前测试分层

### 结构护栏

用于防止架构回退。

例如：

- 宿主不再直连 raw turn
- 宿主不再直连 raw runtime registry
- 关键文件保持单一职责

### 行为回归

用于保护用户看得见的能力。

例如：

- one-shot closeout
- interactive 行为
- Telegram / 微信 `/stop`
- 文件回传
- visible events

### 真相源回归

用于保护 session、checkpoint、SQLite ledger 和配置入口不被旁路替代。

## 当前规则

1. 先写失败测试，再写实现。
2. 新文档路径和新模块边界如果会影响测试，测试要一起迁移。
3. 文档驱动和测试驱动要一起工作，不能一个更新、另一个失效。
