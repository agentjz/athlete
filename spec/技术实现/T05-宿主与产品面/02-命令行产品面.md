# 命令行产品面

## 文档目的

本文说明 CLI 当前的产品面结构，以及 fast path、lightweight path 和 full runtime path 的分工。

## 模块目标与当前状态

CLI 当前负责把常用命令分层启动，避免轻命令拉起整套 runtime，同时保证 one-shot 和 interactive 仍走统一宿主边界。

## 正式边界

主要代码边界：

- `src/cli.ts`
- `src/cli/program.ts`
- `src/cli/commands/`
- `src/cli/runtime.ts`
- `src/cli/oneShot.ts`
- `src/ui/streamRenderer.ts`
- `src/ui/streamRenderer/`
- `src/ui/todoStyling.ts`

## 主路径

当前 CLI 启动链分为三类：

- fast path：`--version`、`help`、轻量信息命令
- lightweight path：不需要完整 runtime 的配置与诊断命令
- full runtime path：one-shot、interactive、宿主服务等重路径

## 模块职责与非职责

### CLI 产品面负责

- 按命令类别组织启动成本
- 暴露产品级命令入口
- 在需要时转入统一 runtime 或 host 路径
- 统一终端可见输出策略（工具调用、dispatch 回执、工具结果成功/失败收据、todo 标记着色）

### CLI 产品面不负责

- 定义核心真相源
- 绕开 `src/host/` 直接执行业务 turn
- 继续长成单个总控大文件

## 当前落地决定

CLI 当前已经固定为“轻命令快速返回，重命令走正式 runtime”的产品面结构。
当前终端渲染默认规则为：模型工具调用显示 `[tool]`；机器层真实启动 teammate / subagent / background execution 后显示 `[dispatch]`；工具结果只显示统一 `成功/失败` 收据。非 `todo_write` 不再展开 preview，失败原因压到同一行；`todo_write` 预览保留 `[ ]/[>]/[x]` 语义并在终端展示，因为它是任务仪表盘，不是原始证据。完整工具结果继续交给工具消息、落盘结果和 tracked 证据链承载。
