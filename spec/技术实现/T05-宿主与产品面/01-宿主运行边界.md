# 宿主运行边界

## 文档目的

本文定义 `src/host/` 的职责，说明宿主如何进入核心、如何绑定 session、如何注入 extra tools，以及哪些行为必须在 host 边界统一收口。

## 模块目标与当前状态

`src/host/` 当前是所有宿主进入核心执行系统的正式接入点。CLI、interactive、Telegram 都通过这里接入，不再各自拼接 turn 主路径。

## 正式边界

主要代码边界：

- `src/host/session.ts`
- `src/host/turn.ts`
- `src/host/boundTurn.ts`
- `src/host/toolRegistry.ts`
- `src/host/types.ts`

## 真相源与状态归属

host 边界当前只负责：

- session 绑定
- turn 调用
- host extra tool 注入

host binding 只保存“绑定到哪个 session id”；真正 session 内容仍由 `SessionStore` 管理。host 本身不保存平行运行真相。

## 主路径

### 1. session 主路径

当前通过以下入口处理 session：

- `createHostSession`
- `createPersistedSession`
- `loadLatestSession`
- `ensureBoundSession`
- `persistBoundSession`

当前规则：

- session 缺失时可以创建正式 session 后再绑定
- session 损坏或 schema 错误时必须 fail closed

### 2. turn 主路径

当前统一通过：

- `runHostTurn`
- `runBoundHostTurn`

进入共享 `runManagedAgentTurn(...)` 主路径，并在 host 边界统一记录 observability、abort、pause 和失败结果。

### 3. host extra tool 主路径

当前统一通过 `createHostToolRegistry(...)` 注入：

`host extra tool -> ToolRegistrySource(kind=host) -> runtime registry -> governance -> exposure`

## 模块职责与非职责

### host 负责

- 为宿主准备或恢复 session
- 调用共享 turn 入口
- 统一注入 host extra tool
- 统一处理 pause、abort、error 和 closeout

### host 不负责

- 再次定义 runtime 真相源
- 直接生成第二套 runtime registry
- 在宿主内部私接 raw turn

## 失败路径与异常路径

当前明确处理：

- 绑定到损坏 session 时 fail closed
- host extra tool 注入失败在注册边界显式暴露
- abort、pause、failed 等结果都在 host 边界统一返回
- 宿主不得直接导入 raw `runManagedAgentTurn(...)` 或 raw `createRuntimeToolRegistry(...)`

## 测试与验证

当前主要由以下测试保护：

- `tests/host/host-runtime-boundary.test.ts`
- `tests/tools/extension-platform.test.ts`
- `tests/observability/operator-observability.test.ts`

## 当前落地决定

当前宿主运行边界已经固定为“统一 session + 统一 turn + 统一 host tool 注入”的结构。增加宿主应复用这套边界，而不是各自复制一条运行主路径。
