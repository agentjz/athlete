# 总体架构

## 一句话

Athlete 当前是一套“耐跑主 Agent + 总指挥调度 + 统一控制面 + 多宿主外壳”的系统。

## 当前正式分层

### 1. 核心执行系统

目录：

- `src/agent/`
- `src/orchestrator/`
- `src/tools/`
- `src/mcp/`
- `src/skills/`
- `src/context/`
- `src/config/`
- `src/control/ledger/`
- `src/tasks/`
- `src/team/`
- `src/execution/`
- `src/worktrees/`
- `src/changes/`

职责：

- turn 执行
- 主 Agent 预处理
- 任务拆分与调度
- tool / skill / MCP 接线
- session / checkpoint / verification / acceptance / runtime stats
- 控制面真相源

### 2. 宿主运行边界

目录：

- `src/host/`

职责：

- 共享 session 入口
- 共享 turn 入口
- 共享 runtime tool registry 入口
- 共享 pause / abort / error / closeout 收口
- 宿主扩展点

### 3. 宿主适配层

目录：

- `src/cli/`
- `src/interaction/`
- `src/ui/`
- `src/shell/cli/`
- `src/telegram/`
- `src/weixin/`

职责：

- 输入输出
- transport
- 展示
- typing / visible / delivery
- stop 语义
- commander 和本地命令

规则：

- 宿主只做壳，不定义 runtime 真相。
- 真正执行 turn 时，宿主必须统一经过 `src/host/`。

## 当前系统主轴

Athlete 的主轴不是“让一个 Agent 什么都亲自做”，而是：

1. lead 先理解和组织任务
2. 控制面把任务状态落盘
3. orchestrator 判断下一步由谁来做
4. 执行者完成后再回到 lead 合流
5. 宿主只负责把这套主路径接给不同通道

## 当前关键边界

### 主 Agent 回答

- 当前任务是什么
- 现在该继续、拆分、派工还是等待

### 控制面回答

- 谁在做
- 哪些任务 blocked
- 哪些后台任务和 worktree 仍有效

### 宿主边界回答

- 一个宿主怎样进入核心
- 它怎样注入 extra tool / callbacks
- 它怎样统一处理 pause / abort / error

### 宿主适配层回答

- 消息从哪来
- 输出到哪去
- 这个宿主的可见语义是什么

## 当前架构约束

1. 控制面真相不允许从宿主层反推。
2. 宿主不允许直接散连 `src/agent/turn.ts` 或 `src/tools/runtimeRegistry.ts`。
3. 调度规则应长在 orchestrator，不应散落在各宿主。
4. tools、skills、MCP、host 扩展都必须通过正式边界生长。
