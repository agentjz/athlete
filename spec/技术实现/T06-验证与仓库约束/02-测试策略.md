# 测试策略

## 文档目的

本文定义仓库当前的测试目标、测试分层和收口标准。

## 当前优先保护对象

1. Lead 的耐跑与恢复能力
2. 调度、派发、等待和合流语义
3. 会话现场、控制面账本和配置入口
4. 宿主运行边界
5. 工具协议和扩展边界
6. 宿主行为回归

## 当前测试分层

### 1. 结构护栏

用于防止架构回退，例如：

- 宿主不得直连底层 turn
- 宿主不得直连 raw runtime registry
- 关键目录与关键文件保持职责清晰

### 2. 真相源回归

用于保护正式状态边界，例如：

- session snapshot
- checkpoint / verification / acceptance
- SQLite control plane
- 配置入口

### 3. 行为回归

用于保护用户可见能力，例如：

- one-shot closeout
- interactive 行为
- Telegram `/stop`
- 可见输出与文件回传
- 工具使用主路径

### 4. 链路验证

用于证明关键模块在真实主路径上可工作，例如：

- provider 接入链路
- host turn
- execution lifecycle
- observability 旁路记录

## 当前规则

1. 先写失败测试，再写实现。
2. 模块边界变动时，测试要同步迁移。
3. 文档驱动和测试驱动必须同步工作。
4. 真相源、调度和宿主边界的改动必须有对应保护测试。

## 当前落地决定

当前仓库把测试视为结构护栏，不把测试当覆盖率装饰。判断是否收口，优先看主路径是否被保护，而不是只看单点断言是否通过。
