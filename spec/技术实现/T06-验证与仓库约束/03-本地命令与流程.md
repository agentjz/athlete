# 本地命令与流程

## 文档目的

本文记录仓库当前使用的本地命令和标准验证流程。

## 当前核心命令

- `npm.cmd run check`
- `npm.cmd test`
- `npm.cmd run build`
- `node dist\\cli.js`

## 当前标准流程

1. 先同步相关 SPEC。
2. 先补失败测试或边界测试。
3. 再修改实现。
4. 最后跑完整验证。

## 当前约束

- Windows 环境优先使用 `npm.cmd`
- 涉及用户入口时，优先验证 CLI、Telegram 的正式主路径
- 涉及真相源、调度或宿主边界时，不只看单元测试，还要看对应集成路径
