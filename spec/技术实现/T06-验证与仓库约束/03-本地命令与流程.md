# 本地命令与流程

## 文档目的

本文记录仓库当前使用的本地命令和标准验证流程。

## 当前核心命令

- `npm.cmd run sync`
- `npm.cmd run sync:readme-capabilities`
- `npm.cmd run verify:generated`
- `npm.cmd run verify:repo-contracts`
- `npm.cmd run check`
- `npm.cmd test`
- `npm.cmd run audit:tests`
- `npm.cmd run build`
- `node dist\\cli.js`

## 当前标准流程

1. 先确认本次变化属于实现、协议、生成物、文档还是测试结构。
2. 涉及生成物时先改事实源，再跑 `npm.cmd run sync`。
3. 行为变化先补失败测试或边界测试；测试粒度保护边界，不锁死无关细节。
4. 修改实现和 spec。
5. 跑 `npm.cmd run verify:generated`、`npm.cmd run verify:repo-contracts` 和相关测试。
6. 收口前跑 `npm.cmd test`。

## 当前约束

- Windows 环境优先使用 `npm.cmd`
- `verify` 必须保持标准入口：`npm run verify:repo-contracts && npm test`
- `verify:repo-contracts` 会检查生成物漂移，不能绕过 `sync`
- 涉及用户入口时，优先验证 CLI、Telegram 的正式主路径
- 涉及真相源、Lead/机器边界或宿主边界时，不只看单元测试，还要看对应集成路径
- `audit:tests` 是审计，不是门禁；它帮助判断测试是否过胖或过细
