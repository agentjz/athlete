# Model Robustness And Strictness Implementation

## 1. Goal And Boundary

This module keeps five runtime contracts aligned:

1. Tool argument strictness tiers: L0, L1, L2.
2. Provider recovery resource limits and Lead review.
3. Managed continuation slice limits and Lead review.
4. Explicit delegation tools plus deterministic execution boundaries.
5. Delegation wait semantics: Lead suspends, machine waits for execution closeout events, then Lead reviews facts.

This document does not define product copy or ad-hoc prompt glue. Runtime truth must remain in structured state, execution closeout, and verified evidence.

## 2. Tool Strictness

### 2.1 Module Boundary

- `src/capabilities/tools/core/toolArgumentStrictness.ts`
  - Classifies tool argument strictness as L0, L1, or L2.
  - Allows controlled unknown-argument stripping only for L0 paths.
- `src/capabilities/tools/core/toolArgumentContract.ts`
  - Validates required, type, enum, and unknown argument contracts.
- `src/capabilities/tools/core/toolPrepare.ts`
  - Runs parse, strictness, contract validation, and guards in one path.
- `src/capabilities/tools/core/registry.ts`
  - Executes tools only after prepared arguments pass the shared boundary.
- `src/capabilities/tools/core/toolFinalize.ts` and `src/types.ts`
  - Persist strictness tier and stripped-unknown warnings in protocol metadata.

### 2.2 Behavior Contract

- L0 strips unknown arguments first, then still blocks required, type, and enum violations.
- L1 and L2 block unknown arguments by default.
- Failures return `INVALID_TOOL_ARGUMENTS` with structured `details`.

## 3. Provider Recovery Resource Limit

### 3.1 Module Boundary

- `src/agent/recoveryBudget.ts`
  - Resolves and evaluates retry attempts plus elapsed time.
- `src/agent/runTurn.ts`
  - Converts exhausted recovery into `pause.provider_recovery_budget_exhausted`.
- `src/agent/turn/managed.ts`
  - Rebounds Lead turns with runtime facts instead of terminating the managed loop directly.
- `src/agent/runtimeTransition/builders.ts`
  - Builds structured recovery transitions.

### 3.2 State Contract

Provider recovery exhaustion writes structured checkpoint state:

- `reason.code = pause.provider_recovery_budget_exhausted`
- `attemptsUsed/maxAttempts/elapsedMs/maxElapsedMs/lastError`
- Lead reviews and chooses the next move.
- Non-Lead execution keeps a visible pause closeout.

## 4. Managed Continuation Slice Limit

### 4.1 Module Boundary

- `src/agent/turn/managedBudget.ts`
  - Resolves and evaluates managed slice count plus elapsed time.
- `src/agent/turn/managed.ts`
  - Writes `pause.managed_slice_budget_exhausted` and preserves the current objective frame.

### 4.2 State Contract

Managed slice exhaustion writes structured checkpoint state:

- `reason.code = pause.managed_slice_budget_exhausted`
- `slicesUsed/maxSlices/elapsedMs/maxElapsedMs`
- Lead does not treat this as final task completion.

## 5. Delegation And Subagent Resource Governance

### 5.1 Module Boundary

- `src/capabilities/tools/packages/tasks/taskTool.ts`
  - Lets Lead explicitly launch subagent executions.
- `src/capabilities/tools/packages/team/spawnTeammateTool.ts`
  - Lets Lead explicitly create teammate executions.
- `src/capabilities/tools/packages/background/backgroundRunTool.ts`
  - Lets Lead explicitly start background command executions.
- `src/execution/boundary.ts`
  - Defines runtime and idle boundaries for subagent, teammate, and background executions.
- `src/execution/agentBoundary.ts`
  - Enforces runtime and idle boundaries in the machine layer.
- `src/agent/turn/delegatedWorkWait.ts`
  - Waits for execution closeout events; wake signals are not truth sources.

### 5.2 Behavior Contract

- Keywords and capability availability never create delegation by themselves.
- There is no machine delegation advice, mode, or policy gate that replaces Lead judgment.
- Machine config does not contain delegation modes.
- Subagents must not be stopped by low tool-call or low model-turn limits during normal work.
- Subagents return to Lead on completion, execution boundary, user interruption, real provider/tool failure, or context/cost boundary once those boundaries have real evidence.
- Execution closeout is the truth source. Wake signals only wake the Lead to read the truth source.
- Lead suspends after dispatch and does not spend model turns polling delegated work.

## 6. Background Termination Idempotency

### 6.1 Module Boundary

- `src/execution/background.ts`
  - Returns no-op for terminal executions.
- `src/capabilities/tools/packages/background/backgroundTerminateTool.ts`
  - Returns `already_terminal/idempotent` without rewriting terminal truth.

### 6.2 Behavior Contract

- Terminal executions can be terminated again without error.
- Terminal truth is not rewritten.
- Running executions still terminate and close out normally.

## 7. Acceptance

The implementation is acceptable only when:

- Tool strictness follows L0/L1/L2 contracts.
- Provider recovery and managed continuation use structured resource-limit transitions.
- Delegation waits are event-driven at the machine layer and do not consume Lead model turns while idle.
- Subagents are not stopped by low tool-call or model-turn budgets.
- Execution boundaries produce structured closeout and return to Lead review.
- Specs, tests, and implementation describe the same runtime behavior.
