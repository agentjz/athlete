# 配置系统

## 文档目的

本文定义 Deadmouse 当前的配置入口、内部拆分、升级边界和宿主共享规则。

## 模块目标与当前状态

配置系统当前只有一个对外入口：`src/config/store.ts`。所有宿主和核心模块都从这里拿 runtime config。

## 正式边界

主要代码边界：

- `src/config/store.ts`
- `src/config/agentModels.ts`
- `src/config/fileStore.ts`
- `src/config/runtime.ts`
- `src/config/env.ts`
- `src/config/schema.ts`

## 真相源与状态归属

当前配置来源包括：

- 全局 `config.json`
- 项目 `.deadmouse/.env`
- 环境变量覆盖

这些来源由 `runtime.ts` 归一后，产出唯一 runtime config。

## 主路径

当前主路径如下：

1. `fileStore.ts` 负责配置文件读写与 schema 升级。
2. `env.ts` 负责 `.env` 装载。
3. `runtime.ts` 合并文件配置、项目 `.env` 与环境变量。
4. `agentModels.ts` 从归一后的默认 provider bundle 与角色级 `.env` 覆盖生成 `lead / teammate / subagent` 三套模型配置。
5. `store.ts` 作为唯一对外入口导出配置能力。

角色级 provider bundle 使用以下环境变量：

- `DEADMOUSE_LEAD_PROVIDER / DEADMOUSE_LEAD_API_KEY / DEADMOUSE_LEAD_BASE_URL / DEADMOUSE_LEAD_MODEL`
- `DEADMOUSE_TEAMMATE_PROVIDER / DEADMOUSE_TEAMMATE_API_KEY / DEADMOUSE_TEAMMATE_BASE_URL / DEADMOUSE_TEAMMATE_MODEL`
- `DEADMOUSE_SUBAGENT_PROVIDER / DEADMOUSE_SUBAGENT_API_KEY / DEADMOUSE_SUBAGENT_BASE_URL / DEADMOUSE_SUBAGENT_MODEL`

任一角色字段缺失时，只回退到默认 `DEADMOUSE_PROVIDER / DEADMOUSE_API_KEY / DEADMOUSE_BASE_URL / DEADMOUSE_MODEL` 的对应字段。角色模型配置只决定当前 agent loop 调用哪个 provider，不改变 Lead 唯一主控规则。

## 模块职责与非职责

### 配置系统负责

- 统一读取和归一化配置
- 维护 schema version
- 为 `lead / teammate / subagent` 输出角色级 provider bundle
- 向 provider、Telegram、MCP 等模块输出共享 runtime config

### 配置系统不负责

- 在宿主内部重复读取 `.env`
- 维护第二套临时配置入口
- 根据任务内容自动决定哪个模型接管主控
- 为历史配置保留长期脏兼容

## 失败路径与异常路径

当前明确处理：

- `schemaVersion` 不支持时 fail closed
- 历史配置只做一次性升级
- 宿主、CLI、tool、service 不得各自读 `.env` 形成第二入口

## 测试与验证

当前主要由以下测试保护：

- `tests/provider-runtime-config.test.ts`
- `tests/agent-role-model-config.test.ts`
- `tests/cli-product-engineering.test.ts`

## 当前落地决定

当前配置系统已经固定为“单一入口 + schema version + runtime 归一化”的结构。增加配置项必须继续沿这条路径进入。
