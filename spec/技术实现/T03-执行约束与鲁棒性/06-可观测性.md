# 可观测性

## 文档目的

本文描述 Deadmouse 当前的 observability 旁路观察面，说明它记录什么、不记录什么、如何与正式真相源分离。

## 模块目标与当前状态

observability 当前只承担运维、排障和事后分析职责，不承担正式裁决职责。

## 正式边界

主要代码边界：

- `src/observability/writer.ts`
- `src/observability/schema.ts`
- `src/observability/report.ts`
- `src/observability/doctor.ts`
- `src/observability/crashRecorder.ts`
- `src/observability/terminalLog.ts`

## 真相源与状态归属

当前必须区分两层状态：

1. `runtimeStats`
   - 进入 session 现场
   - 属于正式运行摘要
2. `.deadmouse/observability/`
   - 只做旁路事件与 crash 记录
   - 不参与正式裁决

当前明确约束：

- observability 不是正式真相源
- JSONL 事件不能反向决定 session、execution、task 或 worktree 的真相
- 写 observability 失败不能改变正式执行语义
- crash recorder 只记录，不吞异常

## 主路径

当前 observability 主路径如下：

- `src/host/turn.ts` 记录 host turn 生命周期
- `src/agent/api.ts` 记录 model request 生命周期
- `src/agent/runTurn.ts` 记录 tool execution 生命周期
- `src/execution/worker.ts` 与 `src/execution/closeout.ts` 记录 execution lifecycle
- `src/telegram/service.ts` 记录宿主服务侧最小运维事件
- `src/observability/terminalLog.ts` 按 session 追加记录交互终端现场；assistant 文本保留，工具调用和工具结果只保留工具名与负载大小摘要

当前落盘约定：

- `.deadmouse/observability/events/YYYY-MM-DD.jsonl`
- `.deadmouse/observability/crashes/<timestamp>-<pid>.json`
- `.deadmouse/observability/terminal/YYYYMMDD/<session-id>.log`

## 模块职责与非职责

### observability 负责

- 旁路记录关键运行事件
- 写 crash 报告
- 镜像交互终端输入与输出
- 为 operator 提供 `doctor observability` 视图

### observability 不负责

- 替代 session、ledger 或 config 成为真相源
- 修改正式执行结果
- 承载宿主展示文案
- 额外读取 env、prompt 或配置来补全终端没有出现的内容

## 失败路径与异常路径

当前明确处理：

- observability 写入失败不改变正式结果
- crash 写盘失败不遮蔽原 crash
- 宿主服务事件只记录最小运维事件，不私造第二条 turn 主路径
- terminal log 只记录降噪后的交互终端现场，不反向驱动调度、收口或恢复

## 当前落地决定

当前 observability 已经固定为“正式真相源之外的结构化旁路观察面”。它负责记录，不负责裁决。
