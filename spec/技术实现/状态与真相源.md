# 状态与真相源

## 规则

Athlete 当前只有一套正式真相源，不允许宿主、提示词或临时 JSON 再发明平行状态。

## 当前正式真相源

### 1. session 现场

介质：

- `paths.sessionsDir/*.json`

核心内容：

- 消息历史
- todoItems
- taskState
- checkpoint
- verificationState
- acceptanceState
- runtimeStats

对应实现：

- `src/agent/session/`
- `src/agent/checkpoint.ts`
- `src/agent/runtimeMetrics.ts`
- `src/agent/verification/`

### 2. 控制面账本

介质：

- `.athlete/control-plane.sqlite`

核心内容：

- task
- team member
- protocol request
- coordination policy
- execution
- worktree

对应实现：

- `src/control/ledger/`
- `src/tasks/`
- `src/team/`
- `src/execution/`
- `src/worktrees/`

### 3. 统一配置入口

入口：

- `src/config/store.ts`

负责：

- 全局配置文件
- 项目 `.athlete/.env`
- 环境变量覆盖
- Telegram / 微信 / MCP / provider runtime config

### 4. 宿主绑定状态

宿主可以持久化自己的绑定和游标，但这些不是控制面裁决源。

例如：

- Telegram session map / offset
- 微信 session map / sync buf / context_token / delivery queue

它们只负责通道运行，不负责定义任务和调度真相。

## 明确不是正式真相源的东西

- prompt 文案
- 可见事件输出
- shell 展示缓存
- JSONL 审计流
- Telegram / 微信的展示消息
- host runtime facade 自己的返回对象

## 当前裁决顺序

1. 先看控制面账本和 session 现场
2. 再看由它们派生的 runtime / orchestration 语义
3. 最后才看宿主展示和可见输出

如果三者冲突，以前两者为准。
