# 状态与真相源

## 文档目的

本文定义 Deadmouse 当前的正式真相源、非真相源对象、裁决顺序和升级边界。

## 总体规则

Deadmouse 当前只允许一套正式真相源体系。宿主缓存、提示词、展示输出和旁路日志都不能成为平行状态来源。

## 当前正式真相源

### 1. 会话现场

存储介质：

- `paths.sessionsDir/*.json`

正式读取边界：

- `src/agent/session/snapshot.ts`

正式内容：

- `messages`
- `createdAt / updatedAt / cwd / title`
- `todoItems`
- `taskState`
- `checkpoint`
- `verificationState`
- `acceptanceState`
- `runtimeStats`
- `sessionDiff`

当前说明：

- 每个 session snapshot 都带 `schemaVersion`
- `checkpoint.flow.pendingToolCalls`、`checkpoint.flow.runState` 和 `checkpoint.flow.compactionRecovery` 属于正式机器状态
- 工具输出中的 protocol/process/collaboration contract 以 message content 形式进入会话现场，不额外引入平行状态文件
- JSON 损坏、schema 不支持或关键字段损坏时，读取边界 fail closed

### 2. 控制面账本

存储介质：

- `.deadmouse/control-plane.sqlite`

主要实体：

- task
- team member
- protocol request
- coordination policy
- execution
- worktree

主要实现：

- `src/control/ledger/`
- `src/tasks/`
- `src/team/`
- `src/execution/`
- `src/worktrees/`

当前约束：

- execution lifecycle 继续承担唯一执行裁决；`background_terminate` 这类动作也必须通过 execution closeout 写回 `aborted`，不能绕过 ledger

### 3. 统一配置入口

对外唯一入口：

- `src/config/store.ts`

当前负责：

- 全局 `config.json`
- 项目 `.deadmouse/.env`
- 环境变量覆盖
- provider、Telegram 和 MCP 的 runtime config

内部拆分：

- `src/config/fileStore.ts`
- `src/config/runtime.ts`
- `src/config/env.ts`

但对外仍然只有一个正式入口。

### 4. 宿主绑定状态

宿主可以保存自己的绑定和游标，但这些只服务宿主运行，不负责控制面裁决。

当前典型对象：

- Telegram session map / offset

当前约束：

- 宿主绑定只能指向正式 session id
- session 缺失时可以重建正式 session 后再绑定
- session 损坏或 schema 错误时必须暴露错误，不能偷偷重建

## 明确不是正式真相源的对象

- prompt 文案
- 可见事件输出
- shell 展示缓存
- observability JSONL
- Telegram 展示消息
- 各宿主临时返回对象
- 未落盘的工具 metadata 临时对象

## 裁决顺序

当前按下面顺序裁决状态：

1. 先看控制面账本和会话现场
2. 再看由它们派生的运行语义
3. 最后才看宿主展示和可见输出

如果冲突，前两者优先。

## 升级与失败边界

当前升级边界如下：

1. 控制面账本通过 SQLite migration 和 `user_version` 升级。
2. 配置通过 `schemaVersion` 升级。
3. 会话现场通过 `schemaVersion` 升级。
4. 升级失败必须在正式读取边界报错，不允许宿主或调用方静默兜底。

## 测试与验证

当前真相源边界主要由下面几类测试保护：

- `tests/session-state-foundation.test.ts`
- `tests/control-plane-ledger.test.ts`
- `tests/provider-runtime-config.test.ts`
- `tests/host-runtime-boundary.test.ts`

## 当前落地决定

Deadmouse 当前已经把“会话现场 + 控制面账本 + 统一配置入口”固定为正式真相源体系；宿主状态、旁路日志和展示输出只能围绕它们工作，不能替代它们。
