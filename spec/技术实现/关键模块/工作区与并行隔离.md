# 工作区与并行隔离

## 当前定位

worktree 是并行执行隔离层，不是任务图本身。

## 当前作用

- 给 teammate lane 提供文件隔离
- 让并行改动不互相踩踏
- 把任务和实际工作目录绑定起来

## 当前关键实现

- `src/worktrees/store.ts`
- `src/worktrees/git.ts`
- `src/worktrees/ledger.ts`

## 当前约束

1. 并行改动优先使用 worktree。
2. task 与 worktree 的绑定现实必须来自控制面。
3. removed worktree 不能被当成仍有效的执行车道。
