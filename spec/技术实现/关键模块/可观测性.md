# 可观测性

## 目标

这层只解决一件事：

- 给操作者/运维/排障补上一条稳定、结构化、可 grep、可脚本处理的旁路观察面。

它不是新的控制面，不是新的状态裁决器，也不是新的 session/execution 真相源。

## 真相边界

必须把下面两层分开：

1. `runtimeStats` / `/runtime`
   - 面向 session 内部运行态摘要
   - 落在正式 session 现场里
2. `.athlete/observability/`
   - 面向 operator event log / crash log / recent failure view
   - 只做旁路记录、运营排障、事后分析

明确约束：

- observability 不是正式真相源。
- JSONL 审计流不能反过来决定 session、execution、task、worktree、policy 的真相。
- observability 写入失败不能修改正式执行语义。
- crash recorder 只能“记录然后保持原失败行为”，不能吞掉异常。
- 正式真相读取边界继续保持 fail closed；observability 只能旁路补充，不能覆盖它。

## 正式路径

模块边界长在：

- `src/observability/`

统一从 `src/project/statePaths.ts` 取路径，不允许散落硬编码：

- `observabilityDir`
- `observabilityEventsDir`
- `observabilityCrashesDir`

当前落盘约定：

- `.athlete/observability/events/YYYY-MM-DD.jsonl`
- `.athlete/observability/crashes/<timestamp>-<pid>.json`

为什么按天拆 JSONL：

- 简单、稳定、可读
- 不需要复杂轮转系统
- 足够支持 grep、tail、脚本清理和最近几天排障

## 事件 schema

事件 schema 版本化，当前 `version = 1`。

统一字段：

- `version`
- `timestamp`
- `event`
- `status`
- `host`
- `sessionId`
- `executionId`
- `identityKind`
- `identityName`
- `durationMs`
- `toolName`
- `model`
- `error`
- `details`

要求：

- 不是每类事件都必须填满所有字段。
- `details` 只放摘要和关键结构化字段，不塞大块 tool result 正文。
- 事件必须保持稳定、可读、可脚本处理。

## 正式打点边界

### 1. host turn

边界：

- `src/host/turn.ts`

记录：

- `host.turn started`
- `host.turn completed`
- `host.turn paused`
- `host.turn aborted`
- `host.turn failed`

最少字段：

- `host`
- `sessionId`
- `cwd`
- `durationMs`
- 失败时的错误摘要

### 2. tool execution

边界：

- `src/agent/runTurn.ts`

记录：

- `tool.execution started`
- `tool.execution completed`
- `tool.execution failed`

最少字段：

- `toolName`
- `sessionId`
- `durationMs`
- `changedPathCount`
- `verificationAttempted`
- `verificationPassed`
- 失败时的错误 code / details

### 3. execution lifecycle

边界：

- `src/execution/worker.ts`
- `src/execution/closeout.ts`

记录：

- `execution.lifecycle started`
- `execution.lifecycle completed`
- `execution.lifecycle paused`
- `execution.lifecycle failed`
- `execution.lifecycle aborted`

最少字段：

- `executionId`
- `profile`
- `lane`
- `actorName`
- `taskId`
- `worktreeName`
- `exitCode`
- `statusDetail`

### 4. model request

边界：

- `src/agent/api.ts`

记录：

- `model.request started`
- `model.request completed`
- `model.request failed`

最少字段：

- `provider`
- `configuredModel`
- `requestModel`
- `durationMs`
- `usageAvailable`
- `recoveryFallback`
- 错误摘要

### 5. host service message flow

边界：

- `src/telegram/service.ts`
- `src/weixin/service.ts`

只记录最小运维事件，不深入 UI 细节：

- inbound accepted
- turn started
- outbound reply queued
- outbound file queued（如果适用）
- delivery failure

turn started 仍然统一走 `src/host/turn.ts`，宿主 service 不得私造第二条 turn 执行路径。

## crash log

共享 crash recorder 负责：

- `uncaughtException`
- `unhandledRejection`

实现要求：

- 写 crash 文件到 `.athlete/observability/crashes/`
- 尽量带上 `pid / argv / cwd / host / sessionId / executionId`
- 写盘失败不能遮蔽原 crash
- 原始失败语义必须保留，不做“假装顺滑”

## operator 入口

当前最小查看入口：

- `athlete doctor observability`

它面向操作者输出：

- observability 路径
- 最近事件文件
- 最近 crash 数量
- 最近失败摘要
- 最近最慢的几类事件

目标是快速排障，不是展示内部术语堆栈。
