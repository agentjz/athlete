# P17 扩展靠事件生长

## 原则

扩展不能只靠往 system prompt 里继续堆规则，应该沿着清晰的扩展点生长。

## 为什么

Athlete 未来一定会长：

- skills
- 更多工具
- 更多角色
- 更多运行时通知

如果没有扩展点，主循环和工具层都会越来越胖。

## 在 Athlete 里的含义

当前扩展入口主要有三种：

- tool registry
- skills
- MCP

下一阶段要补的是更明确的 runtime / lifecycle 事件点。

## 当前对应

- `src/tools/runtimeRegistry.ts`
- `src/skills/catalog.ts`
- `src/mcp/`
