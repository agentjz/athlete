# Security

Please report security issues privately before opening a public issue.

Send a clear report with:

- affected version or commit
- reproduction steps
- expected impact
- relevant logs or artifacts with secrets removed

Do not include API keys, tokens, private credentials, or sensitive user data in reports.

This project is experimental. Security fixes are handled case by case, with priority on issues that affect tool execution, credential exposure, filesystem access, MCP integration, or persisted runtime state.
